\begin{abstract}
We demonstrate LLARS, an open-source platform that enables domain experts and developers to collaboratively build, test, and evaluate LLM-based systems.
Developing such systems for sensitive domains requires both domain knowledge---understanding what constitutes quality---and technical skill for prompt design and model configuration, yet these groups typically lack a shared environment.
LLARS addresses this with three integrated modules:
\textbf{Collaborative Prompt Engineering} lets both groups co-author prompts in a real-time editor with version control and instant LLM testing.
\textbf{Batch Generation} systematically produces the Cartesian product of prompts\,$\times$\,models\,$\times$\,data items with cost tracking.
\textbf{Evaluation} supports human and LLM evaluators assessing outputs via rating, ranking, labeling, or comparison with live inter-rater agreement metrics.
Each module operates independently or as part of an end-to-end pipeline.
LLARS is actively used for developing AI-assisted systems in psychosocial counselling, where domain experts and developers jointly engineer prompts, compare models, and evaluate outputs.

Organizations building LLM-based systems for sensitive domains---psychosocial counseling, legal analysis, medical documentation---must write effective prompts, select suitable models, and rigorously evaluate outputs, requiring both domain expertise (understanding \emph{what} constitutes quality) and technical skill (structuring prompts and configuring LLM pipelines).
Yet these two groups typically work in isolation: prompts are iterated in shared documents without version control, and no single tool supports the full cycle from collaborative prompt development through systematic evaluation.

We present LLARS (LLM Assisted Research System), an open-source platform that closes this gap with three integrated modules (Figure~\ref{fig:pipeline}).
\emph{Collaborative Prompt Engineering} lets domain experts and developers co-author prompts in a shared real-time editor with version control and instant LLM testing.
\emph{Batch Generation} systematically produces the Cartesian product of prompts\,$\times$\,models\,$\times$\,data items with cost estimation and budget control.
\emph{Evaluation} lets human and LLM evaluators assess outputs side by side---via rating, ranking, labeling, or pairwise comparison---with live agreement metrics (Krippendorff's~$\alpha$, Cohen's~$\kappa$, and others).
Each module can be used independently with JSON/CSV export, and the platform connects to any OpenAI-compatible provider including locally hosted models.

Table~\ref{tab:comparison} surveys twelve existing platforms along these dimensions.
Agenta~\cite{agenta2024} combines prompt management with LLM-as-judge scoring but lacks real-time co-editing and structured human evaluation with agreement metrics.
Phoenix~\cite{phoenix2024}, Langfuse~\cite{langfuse2024}, and W\&B~Weave~\cite{weave2024} provide dataset experiments and per-output feedback but focus on tracing rather than systematic batch generation or multi-evaluator campaigns.
ChainForge~\cite{arawjo2024chainforge} supports visual multi-model comparison but omits human evaluation.
Label~Studio~\cite{labelstudio2024} and Argilla~\cite{argilla2024} offer structured human assessment---Label~Studio also computes basic agreement---but require externally generated outputs, disconnecting evaluation from prompt development.
LangSmith~\cite{langsmith2024}, Braintrust~\cite{braintrust2024}, Maxim~\cite{maxim2024}, and Vellum~\cite{vellum2024} link prompting to evaluation but are closed-source, lack collaborative editing, and omit batch generation as a pipeline stage.

LLARS is the first open-source platform to unify collaborative prompt engineering, Cartesian-product batch generation, and hybrid human--LLM evaluation with agreement analytics, enabling developers and domain experts to work together from prompt development through rigorous assessment.


\section{Platform Overview}

LLARS runs as a containerized web application and connects to any OpenAI-compatible LLM provider, including commercial services (OpenAI, Anthropic, Google) and locally hosted models via Ollama.
Teams register providers with their own API keys; registered models then appear throughout the platform---in prompt testing, batch generation, and as LLM evaluators.

\subsection{Collaborative Prompt Engineering}

The prompt engineering module (Figure~\ref{fig:prompt-editor}) lets multiple users edit the same prompt simultaneously in a real-time collaborative editor built on Yjs~\cite{jahns2023yjs}.
Prompts are structured as reorderable message blocks---system instructions, few-shot examples, user templates---with \texttt{\{\{variable\}\}} placeholders that the platform extracts into a shared palette for the team.
While editing continues, any collaborator can test the current prompt against a selected LLM: the response streams back instantly, and multiple team members can test different models or sample data in parallel.
A version control system tracks every saved revision with multi-author attribution, supporting comparison and rollback.
Finalized prompts are exported as JSON or passed directly to batch generation.

\subsection{Batch Generation}

Once prompts are ready, the batch generation module (Figure~\ref{fig:batch-generation}) produces outputs at scale.
A guided wizard walks users through three steps: selecting a data source (an existing evaluation scenario, an uploaded CSV/JSON file, or no source for self-contained prompts), choosing which prompts and models to combine, and setting budget limits.
LLARS then generates the full Cartesian product---$N$~data items $\times$ $M$~prompts $\times$ $K$~models---matching template variables to data columns automatically.
Before execution begins, the platform estimates costs based on model pricing and token projections; optional budget caps pause jobs that exceed thresholds.
Team members can monitor progress in real time as outputs stream in.
Every result records its full provenance---source item, prompt, model, token count, and cost---and is exportable as CSV or JSON.
Completed batches transfer directly into evaluation through the Scenario Wizard, preserving model attribution for cross-model comparison.

\subsection{Evaluation}

The evaluation module brings human evaluators and LLM evaluators together in a single framework.
Four general evaluation types are available (Figure~\ref{fig:scenario-wizard}): \emph{rating} with configurable multi-dimensional Likert scales and weighted dimensions, \emph{ranking} where evaluators sort items into ordered buckets, \emph{labeling} for binary or multi-class classification, and pairwise \emph{comparison}.
The type system is extensible; LLARS ships with two additional domain-specific types for psychosocial counseling (Section~\ref{sec:counseling}).
An AI-assisted Scenario Wizard guides users through scenario creation in five steps, analyzing uploaded data to recommend the most suitable evaluation type.
Alternatively, scenarios can be created directly from batch generation outputs, preserving model attribution for systematic cross-model comparison.

Scenario owners control how items are distributed---all items to every evaluator for full inter-rater analysis, or configurable subsets for faster throughput.
Where most tools limit LLMs to pairwise judging~\cite{zheng2024judging}, LLARS treats them as full evaluation participants that rate, rank, label, and compare alongside humans using customizable type-specific prompts.
As evaluations come in, agreement metrics update live---Krippendorff's~$\alpha$, Cohen's and Fleiss'~$\kappa$, Kendall's~$\tau$, and ICC---filterable by evaluator pairing (human--human, LLM--LLM, or human--LLM) to pinpoint where judgments diverge.
All data, metrics, and configurations are exportable as JSON or CSV.

\section{Application: Online Counselling}
\label{sec:counseling}

LLARS is actively used in the development of CAIA~\cite{steigerwald2025caia}, an AI-assisted support system for text-based psychosocial counselling that provides counsellors with seven LLM-driven functionalities.
Counselling researchers and developers collaboratively refined the underlying prompts in LLARS, iterating against real counselling data until outputs met professional standards.
For one functionality---automated subject line generation~\cite{steigerwald2025subjectline}---the complete pipeline was exercised: batch generation produced 253~subject lines across 11~LLMs, five counselling professionals and an LLM evaluator rated them (1,518~assessments in total), and the resulting cross-model comparison directly informed the selection of a privacy-preserving open-source model for deployment.

This work also motivated two domain-specific evaluation types (Figure~\ref{fig:scenario-wizard}):
\emph{Mail Rating} for assessing AI-generated counsellor responses on dimensions such as coherence and empathy, and \emph{Authenticity} for classifying counselling threads as real or AI-generated with ground truth labels.
