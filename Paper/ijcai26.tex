\begin{abstract}
We demonstrate LLARS (LLM Assisted Research System), an open-source platform that integrates human evaluation, LLM-as-Judge automation, and retrieval-augmented generation (RAG) for systematic assessment of large language model outputs.
LLARS uniquely combines (1) flexible rating and ranking interfaces for comparative human evaluation, (2) automated pairwise comparison using configurable LLM judges with position bias mitigation, (3) a RAG pipeline with web crawling for grounded evaluation, and (4) real-time multi-user collaboration via conflict-free replicated data types.
The system has been deployed in academic research for evaluating LLM-generated counseling responses, demonstrating its practical applicability.
Our live demonstration showcases the complete evaluation workflow from document ingestion through collaborative annotation to automated analysis.

\vspace{0.5em}
\noindent\textbf{Demo Video:} \url{https://llars-project.github.io/demo}
\end{abstract}

\section{Introduction}

The proliferation of Large Language Models (LLMs) has created an urgent need for systematic evaluation frameworks that combine human judgment with automated assessment~\cite{chang2024survey}.
While benchmarks like MT-Bench~\cite{zheng2024judging} provide standardized metrics, research contexts often require customizable evaluation criteria and collaborative annotation workflows.

Existing tools address parts of this challenge: Label Studio~\cite{labelstudio2024} and Argilla~\cite{argilla2024} offer annotation interfaces but lack integrated LLM evaluation; Chatbot Arena provides pairwise comparison but operates as a public benchmark rather than a customizable research tool.
None combine human evaluation, LLM-as-Judge, RAG, and real-time collaboration in a single platform.

LLARS addresses this gap by providing an integrated environment specifically designed for LLM evaluation research.
The system has been used in published studies on AI-assisted counseling~\cite{steigerwald2025subjectline,steigerwald2025caia}, demonstrating its practical applicability in sensitive domains requiring careful human oversight.

\textbf{Backend:} Python/Flask 3.0 with MariaDB for relational data and ChromaDB for vector embeddings.
LLM integration via LiteLLM supports 100+ model providers.
Socket.IO enables real-time updates across clients.

\textbf{Frontend:} Vue.js 3.4 with Vuetify provides responsive interfaces.
Real-time collaboration uses Yjs~\cite{shapiro2011crdt}, enabling conflict-free concurrent editing with live cursor synchronization.

\textbf{Authentication:} Authentik integration provides OAuth2/OIDC single sign-on with role-based access control (Admin, Researcher, Evaluator, Chatbot Manager).

\section{Key Features}

\subsection{Human Evaluation: Rating and Ranking}

LLARS supports two complementary evaluation paradigms designed for systematic LLM output assessment:

\textbf{Rating Mode:} Annotators assign scores on configurable scales (e.g., 1-5 stars) across multiple quality dimensions.
The interface presents LLM outputs alongside source material, supporting assessments of accuracy, helpfulness, and domain-specific criteria.

\textbf{Ranking Mode:} Annotators order multiple outputs by preference using drag-and-drop interaction.
This comparative approach is effective when absolute standards are difficult to define.

Both modes support:
\begin{itemize}
    \item Time-bounded evaluation periods with progress tracking
    \item Flexible distribution (all items to all annotators vs.\ round-robin)
    \item Randomized presentation order to minimize bias
    \item Real-time inter-annotator agreement statistics (Krippendorff's $\alpha$)
\end{itemize}

\subsection{LLM-as-Judge Module}

The LLM-as-Judge module implements automated pairwise comparison following established protocols~\cite{zheng2024judging}:

\textbf{Configurable Criteria:} Researchers define evaluation rubrics through an administrative interface, specifying dimensions like coherence, factuality, or domain-specific quality measures.

\textbf{Position Bias Mitigation:} Each comparison runs twice with swapped positions.
Results are aggregated to reduce the well-documented position bias in LLM judgments.

\textbf{Session Management:} Batch processing with pause/resume capability enables large-scale evaluations.
Real-time progress visualization shows win rates, tie frequencies, and confidence distributions.

\textbf{Multi-Model Support:} Via LiteLLM, any supported model (GPT-4, Claude, Llama, Mistral, etc.) can serve as judge, enabling meta-evaluation of judge reliability.

\subsection{RAG Pipeline with Web Crawling}

LLARS includes an end-to-end RAG pipeline for document-grounded evaluation:

\textbf{Document Ingestion:} Support for PDF, HTML, Markdown, and plain text with automatic chunking and metadata extraction.

\textbf{Web Crawler:} Playwright-based crawler with configurable depth, domain restrictions, and automatic screenshot capture for visual content preservation.

\textbf{Hybrid Retrieval:} Combined semantic search (via embedding models) and lexical search (BM25) with reciprocal rank fusion.
Supports both local models (sentence-transformers) and API-based embeddings.

\textbf{Chatbot Builder:} No-code interface for creating RAG-enhanced chatbots that can be evaluated through the platform's rating system.

\subsection{Real-Time Collaboration}

Built on Yjs CRDTs, LLARS enables true collaborative evaluation:

\textbf{LaTeX/Markdown Editor:} Multi-user editing with live cursor positions, user presence indicators, and integrated PDF compilation.

\textbf{AI Writing Assistant:} Context-aware completions, rewriting suggestions, and RAG-powered citation finding assist collaborative document creation.

\textbf{Version History:} Git-like versioning with diff visualization enables tracking of document evolution.

\section{Application: Counseling Response Evaluation}

LLARS has been deployed for evaluating LLM-generated responses in online mental health counseling~\cite{steigerwald2025caia}.
This use case demonstrates the platform's capabilities in sensitive domains requiring careful human oversight.

\textbf{Setup:} Email threads from an anonymized counseling dataset were processed through LLARS.
Responses generated by GPT-4, Claude-3, and Mistral-7B were evaluated by trained psychology students.

\textbf{Evaluation Design:}
\begin{itemize}
    \item 5 annotators rated responses on 4 quality dimensions
    \item Ranking of alternative responses within each thread
    \item LLM-as-Judge comparison for all response pairs
    \item RAG-enhanced context from counseling guidelines
\end{itemize}

\textbf{Results:} 2,400 ratings and 600 rankings collected over 4 weeks.
Inter-annotator agreement ranged from $\alpha = 0.61$ to $0.78$.
LLM-as-Judge showed moderate correlation ($r = 0.54$) with human rankings, with systematic differences in empathy assessment---insights that informed subsequent model development.

\section{Demonstration Scenario}

Our live demonstration walks through the complete LLARS workflow:

\begin{enumerate}
    \item \textbf{Document Ingestion:} Upload evaluation guidelines and crawl relevant web resources into a RAG collection
    \item \textbf{Scenario Setup:} Create an evaluation scenario with rating dimensions and annotator assignments
    \item \textbf{Human Evaluation:} Demonstrate the rating and ranking interfaces with real-time progress tracking
    \item \textbf{LLM-as-Judge:} Configure and run automated pairwise comparison with position bias mitigation
    \item \textbf{Analysis:} View agreement statistics, compare human vs.\ automated judgments, and export results
    \item \textbf{Collaboration:} Show real-time multi-user editing in the LaTeX workspace
\end{enumerate}

Attendees can interact with the system using prepared example data or upload their own content for evaluation.

\section{Technical Requirements and Availability}

LLARS requires Docker 20.10+, 16GB RAM, and 4+ CPU cores.
GPU acceleration is optional for local embedding models.
The system is available as open-source software under MIT license.
