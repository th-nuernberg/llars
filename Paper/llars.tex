\begin{abstract}
%==============================================================================
Evaluating Large Language Model outputs in specialized domains requires expertise that machine learning researchers often lack, while domain experts lack the technical infrastructure for systematic evaluation.
We present LLARS, an open-source platform enabling interdisciplinary teams to collaboratively evaluate LLM-generated content.
The system provides configurable evaluation workflows including rating scales, bucket-based ranking, and pairwise comparison, with automatic computation of inter-annotator agreement via Krippendorff's~$\alpha$.
LLARS has been deployed in psychosocial online counseling research, supporting studies on automated anonymization and subject line generation.
Beyond evaluation, the platform integrates collaborative prompt engineering, a no-code RAG chatbot builder, and multiple LLM agent reasoning modes.
LLARS is available at \url{https://llars.e-beratungsinstitut.de}.
\end{abstract}

\section{Introduction}
%==============================================================================

The deployment of Large Language Models in sensitive domains such as healthcare, legal advice, and psychosocial counseling requires rigorous evaluation by domain experts \cite{chang2024survey}.
However, current evaluation practices face a fundamental gap: the people best qualified to assess output quality often lack the technical infrastructure to conduct systematic evaluations, while ML researchers lack the domain expertise to judge appropriateness.

This challenge is particularly acute in interdisciplinary research settings.
In our work on AI-assisted psychosocial online counseling, counselors must evaluate whether LLM-generated responses are empathetic, clinically appropriate, and safe---judgments that require years of professional training.
Comparing outputs from different prompting strategies, models, and fine-tuning approaches across hundreds of samples becomes cognitively overwhelming without proper tooling.

Existing annotation platforms such as Label Studio \cite{labelstudio2024}, Argilla \cite{argilla2024}, and Prodigy \cite{prodigy2024} focus primarily on traditional NLP labeling tasks.
While recent updates have added LLM evaluation templates, these tools lack native support for expert-driven comparative evaluation with built-in statistical analysis.
Crowdsourcing platforms like Chatbot Arena \cite{zheng2024judging} emphasize preference collection at scale but are unsuitable for sensitive domains requiring specialized expertise.

We present LLARS (LLM Assisted Research System), a platform designed specifically for interdisciplinary LLM evaluation (Figure~\ref{fig:concept}).
LLARS has been successfully deployed in multiple published studies, including automated subject line generation for e-mental health \cite{steigerwald2025subjectline} and field evaluation of AI-assisted counseling support \cite{steigerwald2025caia}.

An \textbf{Nginx reverse proxy} routes requests to the appropriate services: the Vue.js frontend serves the user interface, the Flask backend handles API requests, and a dedicated \textbf{Yjs server} enables real-time collaboration via WebSockets using Conflict-free Replicated Data Types (CRDTs) \cite{shapiro2011crdt}.

The backend orchestrates four core services: (1)~the \textbf{Evaluation Engine} manages scenarios, annotations, and agreement computation; (2)~the \textbf{LLM Gateway} provides a unified interface to multiple providers (OpenAI, Anthropic, local models) via LiteLLM; (3)~the \textbf{RAG Pipeline} handles document ingestion, chunking, and semantic search; and (4)~the \textbf{Agreement Calculator} computes Krippendorff's~$\alpha$ in real-time.

Data persistence uses \textbf{MariaDB} for relational data and \textbf{ChromaDB} as the vector store for embeddings.
Authentication integrates with \textbf{Authentik} via OAuth2/OIDC, supporting institutional single sign-on with role-based access control.

\section{Core Features}
%==============================================================================

LLARS provides three integrated modules for the complete LLM research lifecycle: evaluation workflows, collaborative prompt engineering, and RAG chatbot development.

\subsection{Evaluation Workflows}

The evaluation module supports three paradigms (Table~\ref{tab:eval-modes}): \textbf{Rating} assigns scores on configurable scales; \textbf{Bucket Ranking} uses a two-phase approach where evaluators first sort samples into quality tiers, then rank within each tier; and \textbf{Pairwise Comparison} presents side-by-side outputs for winner selection.

\begin{table}[t]
\centering
\small
\begin{tabular}{@{}lp{5cm}@{}}
\toprule
\textbf{Mode} & \textbf{Description} \\
\midrule
Rating & Scores on configurable scales (1--5 stars, Likert). Multiple dimensions per scenario. \\
\addlinespace
Bucket Ranking & Sort into quality tiers first, then rank within tiers. Reduces cognitive load. \\
\addlinespace
Pairwise & Side-by-side comparison with winner selection and optional justification. \\
\bottomrule
\end{tabular}
\caption{Evaluation paradigms supported by LLARS.}
\label{tab:eval-modes}
\end{table}

All modes automatically compute Krippendorff's~$\alpha$ \cite{krippendorff2011computing} to quantify inter-annotator agreement.
Results are visualized in real-time dashboards, enabling researchers to monitor progress and identify ambiguous samples.

\subsection{Collaborative Prompt Engineering}

Multiple team members can simultaneously edit prompts with real-time synchronization.
Each user is assigned a distinct color, and a simplified Git-based version control maintains complete history.
Prompts can be tested against configured LLM backends directly, with outputs immediately available for comparative evaluation.

The platform follows an \textbf{AI-by-Design} philosophy: at every text input, users can invoke LLM assistance for rephrasing, grammar correction, or content suggestions.

\subsection{RAG Chatbot Wizard}

The no-code \textbf{RAG Chatbot Wizard} enables domain experts to create knowledge-grounded chatbots without programming.
Users provide a URL, and LLARS automatically crawls the content, generates embeddings, and deploys a Retrieval-Augmented Generation chatbot.

LLARS also supports multiple \textbf{agent reasoning modes}: Act (direct execution), ReAct \cite{yao2023react} (interleaved reasoning and acting), and Reflect (self-critique before response), which can be compared side-by-side.

\section{Demonstration}
%==============================================================================

The live demonstration showcases a complete evaluation workflow from the psychosocial counseling domain:

\begin{enumerate}
\item \textbf{Scenario Creation}: We configure an evaluation scenario with rating dimensions for empathy, appropriateness, and helpfulness.
\item \textbf{Data Import}: LLM outputs from different prompting strategies are imported for comparative evaluation.
\item \textbf{Live Evaluation}: Attendees experience the bucket-based ranking interface and observe real-time Krippendorff's~$\alpha$ computation as annotations arrive.
\item \textbf{RAG Deployment}: We deploy a domain-specific chatbot from web content in under two minutes using the no-code wizard.
\end{enumerate}

A demonstration video is available at \url{https://llars.e-beratungsinstitut.de/demo}.

\section{Conclusion}
%==============================================================================

LLARS enables interdisciplinary teams to conduct rigorous LLM evaluations by providing domain experts with professional-grade tooling.
The platform has been validated in psychosocial online counseling research and is designed for extension to other sensitive domains requiring expert judgment.
LLARS is open-source and available at \url{https://llars.e-beratungsinstitut.de}.
